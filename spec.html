単一ファイルSPA採用の理由

🚀
デプロイ簡易性
GitHub PagesにPushするだけで即反映。ビルドステップ・CI不要。
🔗
ルーティング不要
SPAのページ切り替えはJS制御。サーバーサイドルーティング設定が不要。
📦
依存ゼロ
React/Vue/Vite等のフレームワーク・バンドラー不使用。Google Fonts CDNのみ。
⚡
即時ロード
ネットワークリクエストが最小。モバイル・低速回線でも高速表示。

function nav(id) {
  // 全 .page から active を外す
  document.querySelectorAll('.page').forEach(p => p.classList.remove('active'));
  // 対象ページを表示
  document.getElementById('pg-' + id)?.classList.add('active');
  // topbar のタイトル・ボタンを更新
  document.getElementById('ttitle').innerHTML = pages[id].t + `<span>` + pages[id].s + `</span>`;
  document.getElementById('tbtn').textContent = pages[id].b;
}

📆 API

自社予約API / Webhook設計

項目仕様
通信方式REST API（同期）＋ Webhook（非同期通知）
認証OAuth 2.0（Client Credentials）＋ JWT（有効期限15分）
Webhookイベントbooking.created / booking.updated / booking.canceled
Webhook署名検証
# 署名生成（送信側）
signature = HMAC-SHA256(secret, timestamp + "." + raw_body)
header: X-Signature: sha256={signature}

# 検証（受信側）
1. X-Signature ヘッダーから signature を取得
2. 同じ計算式で expected_signature を生成
3. hmac.compare_digest() で定数時間比較

# リプレイ攻撃防止
- timestamp が ±5分超の場合は拒否
- event_id の重複チェック（冪等性）→ 重複は無視

# リトライ
- 失敗時: 指数バックオフ（1s → 2s → 4s → 8s ...）

// OK: display:none（完全に非表示・レイアウトに影響しない）

// CSS初期状態

// goStep(): 位置計算後に表示（チラつき防止）

// positionStep 内で最後に:
wrap.style.display = 'flex';  // 表示

// endTour(): 完全非表示＋スタイルリセット
wrap.style.display   = 'none';
wrap.style.top       = '';
wrap.style.left      = '';
wrap.style.transform = '';

位置計算ロジック

JavaScript
function positionStep(step) {
  const isNarrow = window.innerWidth < 1100;  // iPad以下は中央固定

  if (!step.target || step.pos === 'center' || isNarrow) {
    // 画面下部中央に固定（モバイル・iPad・Step1）
    wrap.style.bottom    = '24px';
    wrap.style.left      = '50%';
    wrap.style.transform = 'translateX(-50%)';
    wrap.style.display   = 'flex';
  } else {
    // デスクトップ: ターゲット要素の右横
    const r = el.getBoundingClientRect();
    // 画面右端にはみ出す場合は左側に反転
    let bx = r.right + 12;
    if (bx + wrapW > vw - 12) bx = r.left - wrapW - 12;
  }
}
💡 isNarrow の閾値を1024px → 1100pxに設定。iPad（1024px）でも確実に中央表示になるよう対応済み。
🔐 SEC

セキュリティ仕様

項目仕様根拠
APIキー管理KMS / Secret Manager格納（DB平文保存禁止）漏洩リスク最小化
通信暗号化TLS 1.2以上必須中間者攻撃防止
Webhook署名HMAC-SHA256（timestamp + body）なりすまし防止
リプレイ防止±5分超拒否 + event_id重複排除同一リクエスト再利用防止
JWT有効期限15分（予約API認証トークン）トークン漏洩時の被害最小化
監査ログ全操作（SoT更新・同期実行・API設定変更）を記録コンプライアンス・追跡可能性
⚠️ GitHub Pagesはパブリックリポジトリ前提の静的ホスティングです。本番APIキー・認証情報は絶対にHTMLにハードコードしないでください。
📚 TERMS

用語集

SoT
Single Source of Truth
正しいデータの唯一の情報源。この画面で管理する店舗情報がSoT。
NAP
Name / Address / Phone
MEO最重要の3情報。全ディレクトリで一致が必要。
engine_owner
Write Permission Controller
NAP情報の書き込み権限を持つサービス。本システムではUberall。
Listing
Directory Listing
各ディレクトリ（Google/Yahoo/ぐるなびなど）への店舗情報掲載エントリ。
AIO
AI Overviews
Google検索のAI概要機能。MEO施策でのAIO対策が重要。
JSON-LD
JSON Linked Data
構造化データの記述形式。LocalBusiness typeでGoogleに店舗情報を正確に伝える。
ULPM
Unified Location Profile Manager
本システムのデータ基盤。SoTの一元管理とAPI連携の橋渡し。
HMAC
Hash-based Message Authentication Code
Webhookの署名方式。SHA-256を使用。
QPM
Queries Per Minute
API呼び出し数制限。差分取得・バッチ設計で対処が必要。
UTM
Urchin Tracking Module
GA4流入計測パラメータ。utm_source/medium/campaignなど。
MEO
Map Engine Optimization
Googleマップ等のローカル検索最適化。SEOのローカル版。
SPA
Single Page Application
単一HTMLファイルでページ遷移を擬似的に実現するアーキテクチャ。
